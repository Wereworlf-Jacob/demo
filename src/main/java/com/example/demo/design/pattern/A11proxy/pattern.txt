代理模式有很多种形式，其中一个一般模式是远程代理模式

比如，在网络上有一个对象是GumballMachine。
我们在本地去访问这个对象是要经过网络的。那么解耦出来，我们在本地生成一个“代表”
这个代表就表示了远程的这个对象。
那么我们本地只需要访问本地代表就可以了。

              interface Subject
              ^                     ^
              |                     |
    realSubject    <----subject---- Proxy

    真正的对象和代理对象都实现同一个接口。由proxy来创建和访问realSubject。
    所以Proxy是持有一个subject的。我们通过本地proxy来访问控制realSubject。这个就是远程代理

虚拟代理：比如我们在请求Swing窗口组件，添加一个CD封面时，因为请求这个CD封面，经过网络加载。
我们需要在加载的时候，如果icon还没有加载进来，那么我们就显示“CD加载中，请稍候”。等到icon加载完毕后
我们将调用icon的paintIcon来显示图象

代理模式可以让我们：将请求分发到远程主题；给创建开销大的对象提供代表；提供某些级别的保护等等。
